Transient 2

Story

Building off my experience with creating Transient 1, I am completely revamping the game. In order to appeal to more casual gamers, the movement of the ball will be controlled by accelerometers, level design will include more varied tilesets, and art will be co-produced with a graphic designer. In production currently, I have began by building a desktop tile and level editor and tester. The basic game structure has been already ported to Android; all classes and structures are fully compatible on the desktop and phone versions.

Highlights

  • Written in Java using the Android SDK.
  • Level, tile, and user data stored in SQLite database.
  • Revamped physics and collision detection. Uses physics, geometry, and linear algebra to create fluid movement over various geometrical shapes.
  • On the fly texture and collision vector adjustment using transformation matrices.
  • Level/Tile editor allows for graphic designer to import textures and test ideas quickly.
  • Realtime game loop uses Model, View, Controller structure.
  • Abstract structures allow easy addition of new tile-types.

Ideas for Future

  • Handle entity spawning for dynamic tiles (e.g. enemy creatures or powerups.
  • Data structure for moving, twisting walkways.
  • Speed and memory optimization for Android.
  • ...much, much more....
